什么时候用 DiD
AB 实验验证
实验组 vs 对照组，验证因果效应而非相关性
非随机分组
按行为/特征划分，两组基线趋势本就不同
策略效果归因
排除季节、趋势等时间因素，剥离净效应
指标下降溯源
验证某行为路径是否真实导致了指标变化
两种分析模式
Standard
标准 DiD
随机分组 / AB 实验
平行趋势假设成立
基准期 → 观测期（2段）
Trend
趋势校正 DiD
自选择分组 / 非随机
主动估计并校正趋势差异
A → B → C → D（4段）
trend 模式估计量：
τ = diff2 − diff1
  = [(D−C)trt−(D−C)ctrl] − [(B−A)trt−(B−A)ctrl]
核心原理
DiD 估计量
τ = Δtrt − Δctrl
Δ = post_mean − pre_mean
精确 Panel SE（有协方差时）
Var(τ) = Var(Δtrt)/nt + Var(Δctrl)/nc
Var(Δ) = Var(post) + Var(pre) − 2·Cov(pre,post)

关键技术决策
⚖️
固定分母设计 — 样本量固定为实验队列总人数，无行为用户贡献 0，避免活跃度结构变化污染跨期比较
📐
精确 Panel SE — SQL 直接输出真实协方差，利用个体跨期相关性精确计算标准误，避免独立假设高估 SE
🔄
趋势校正 vs 平行趋势 — trend 模式不依赖平行趋势假设，而是主动估计并吸收趋势差异，自选择分组更可信
📊
业务量化 — 自动输出"人均效应 × 用户数 = 日均总效应"，百分比 + 绝对量双维度，决策层可读
🔗
向后兼容 — 旧字段名 / 旧模式名自动静默映射，协方差缺失时自动退化为独立假设近似
